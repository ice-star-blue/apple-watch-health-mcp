# 隐私说明

健康数据属于高度敏感信息。部署和使用本项目代表你选择把部分 HealthKit 数据从 Apple 设备发送到自己控制的 Cloudflare Worker。

## 会处理的数据

iPhone App 在获得系统授权后，可能读取并上传以下项目的最近一次记录：

- 心率、静息心率、步行平均心率、心率变异性（HRV）
- 血氧、呼吸频率、最大摄氧量（VO2 Max）
- 步数、步行/跑步距离、活动能量、基础能量、站立时间、锻炼时间
- 环境噪音暴露、耳机音量暴露
- 步行速度、步长、六分钟步行距离
- 最近一次睡眠片段

Watch App 只在用户主动启动实时模式后上传实时心率、采样时间和设备来源标签。

## 数据流与保存

- App 使用 HTTPS 把数据发送到配置的 Worker。
- Worker 的 Durable Object 合并并保存每项指标的最新快照。
- 新数据会覆盖同一指标的旧快照，但其他未更新指标会继续保留。
- 本项目不包含用户账户、分析 SDK、广告 SDK或历史数据库。
- Cloudflare 仍可能按照其服务条款处理网络和运行日志。部署者应自行查看 Cloudflare 的隐私与日志设置。

## 谁能读取

- 知道完整 `/mcp/<MCP_PATH_TOKEN>` 地址的人或服务可以读取最新健康快照。
- 知道 `UPLOAD_TOKEN` 的人可以上传数据或删除快照。
- iPhone/Watch App 内的上传令牌最终会存在于安装包中，因此该方案适合个人实验，不应视为高安全等级的多人医疗系统。

两个令牌必须随机生成、彼此不同，并且不能写入源码、截图、Issue 或公开日志。怀疑泄露时应立即在 Cloudflare 中轮换。

## 用户控制

- HealthKit 权限可以在 iPhone 的“设置/健康”中随时撤销。
- 停止 Watch 实时模式后，不再持续采集实时心率。
- 删除两个 `RelayConfig.plist` 后重新安装，App 将不再上传。
- 使用 README 中的 `DELETE /data` 命令可以清空云端快照。
- 删除 Worker 和 Durable Object 可彻底停止此服务。

## 医疗声明

本项目仅用于学习和个人信息展示。数据可能过期、缺失或因系统调度而延迟，不能用于紧急情况、医疗诊断、治疗决定或替代专业医护意见。
